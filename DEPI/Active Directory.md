### المشكلة في الشبكات الصغيرة vs الكبيرة
- لو عندك شركة صغيرة جدًا (5 كمبيوترات و5 موظفين):  
  سهل جدًا تدير كل حاجة يدويًا:  
  - تدخل على كل جهاز لوحده.  
  - تعمل حسابات محلية (local users).  
  - تعدل إعدادات كل موظف (صلاحيات، برامج، إلخ).  
  - لو جهاز عطلان، تروحله بنفسك وتصلحه في مكانه.  
  حياة مريحة وهادية.

- لما الشركة تكبر فجأة (157 جهاز و320 موظف في 4 مكاتب):  
  الطريقة اليدوية دي تبقى **كارثة**:  
  - هتتعب وتضيع وقت رهيب لو عملت يوزر أو سياسة على كل جهاز لوحده.  
  - مستحيل تطبق قواعد أمن موحدة (مثل منع تغيير الـ wallpaper أو منع تنزيل برامج).  
  - الدعم onsite هيبقى مستحيل (مش هتقدر تروح كل مكتب كل يوم).  
  → الحل: **نظام مركزي** بدل الإدارة المبعثرة.

### إيه هو Windows Domain؟
- الدومين ببساطة: مجموعة كبيرة من المستخدمين + الأجهزة (كمبيوترات) تحت إدارة واحدة للشركة كلها.  
- مش كل جهاز لوحده، لكن كلهم مرتبطين ببعض في "نادي" واحد.

### اللي بيدير الدومين ده: Active Directory (AD)
- AD هو **قاعدة بيانات مركزية** (repository) تخزن كل المعلومات المهمة:  
  - حسابات الموظفين (users) وكلمات السر.  
  - الأجهزة (computers).  
  - المجموعات (groups)، الطابعات، المشاركات، إلخ.  
- السيرفر اللي بيشغل AD ده اسمه **Domain Controller** (DC) – ده الـ "مدير النادي" الرئيسي.
### أهم المزايا (اللي بتفرق في الواقع)
1. **إدارة الهويات مركزية (Centralized Identity Management)**:  
   - يوزر واحد + باسوورد واحد يشتغل على **أي** جهاز في الشركة.  
   - مش محتاج تعمل حساب على كل كمبيوتر لوحده.

2. **إدارة السياسات الأمنية (Security Policies)**:  
   - تعمل سياسة مرة واحدة في AD (مثل: كلمة سر قوية، قفل الشاشة بعد 10 دقايق، ممنوع الوصول للـ Control Panel).  
   - السياسة دي تطبق تلقائيًا على كل المستخدمين أو الأجهزة أو مجموعات معينة.
### مثال حقيقي
- في الجامعة أو المدرسة:  
  - بتاخد يوزر وباسوورد واحد.  
  - تدخل من أي كمبيوتر في الكامبس.  
  - الكمبيوتر يروح يسأل الـ AD: "اليوزر ده صح؟ عنده صلاحية؟"  
  - مش بيقدر تغير إعدادات كتير (no admin rights) عشان السياسات معمولة مركزيًا.
يا فاطمة، هشرحلك الجزء ده بالمصري البسيط والواضح، مركز على المعلومات الأساسية المهمة جدًا اللي هتفيدك في الـ TryHackMe room أو في أي شغل/بلوج عن Active Directory. هاخد النقاط الجوهرية وأرتبها عشان تكون سهلة القراءة والفهم، وأضيف شوية نصايح عملية وأدوات/مصادر في النهاية.

### Active Directory Domain Services (AD DS)
- AD DS هو الـ "كتالوج" أو قاعدة البيانات الرئيسية اللي بتحفظ كل حاجة في الشبكة.
- بيخزن objects كتير، أهمها:
  - Users (مستخدمين)
  - Groups (مجموعات)
  - Machines/Computers (الأجهزة)
  - Printers, Shares, وغيرهم
### أنواع objects المهمة

1. **Users**
   - من أكتر الأنواع شيوعًا.
   - يُسمّوا **Security Principals** → يعني يقدر يتعرف عليهم (authenticate) وياخدوا صلاحيات على موارد (files, printers, إلخ).
   - نوعين رئيسيين:
     - **People** → موظفين عاديين.
     - **Service Accounts** → حسابات للخدمات (مثل IIS أو SQL Server)، صلاحياتهم محدودة جدًا على اللي محتاجينه بس.

2. **Machines / Computers**
   - كل جهاز ينضم للدومين بيتم إنشاء object (machine account) له تلقائيًا.
   - ده كمان **Security Principal**، لكن صلاحياته محدودة جدًا.
   - الجهاز نفسه بيبقى local admin على نفسه.
   - اسم الحساب: اسم الجهاز + دولار ($)  
     مثال: جهاز اسمه DC01 → الحساب بتاعه DC01$
   - الباسوورد بتاع machine accounts:
     - طويل جدًا (120 حرف عشوائي تقريبًا).
     - بيتجدد تلقائيًا كل فترة (عادة 30 يوم).

3. **Security Groups (المجموعات الأمنية)**
   - بدل ما تعطي صلاحيات لكل يوزر لوحده، تعطيها للمجموعة كلها → أسهل في الإدارة.
   - المجموعات كمان **Security Principals** → تقدر تاخد صلاحيات على موارد.
   - ممكن تضيف فيها: users + machines + مجموعات تانية.

### الأداة الرئيسية للإدارة: Active Directory Users and Computers
- افتحها من الـ Domain Controller (من Start Menu).
- هتشوف الهيكلية كلها:
  - Organizational Units (OUs) → حاويات تنظيمية.
  - Containers افتراضية (مش OUs حقيقية):
    - Builtin → مجموعات ويندوز الافتراضية.
    - Computers → الأجهزة الجديدة بتروح هنا أول ما تنضم.
    - Domain Controllers → الـ DCs.
    - Users → يوزرز ومجموعات افتراضية.
    - Managed Service Accounts → حسابات خدمات متقدمة.

- في الـ room بتاعت THM:
  - في OU اسمها THM، وتحتها 5 OUs فرعية: IT, Management, Marketing, R&D, Sales.
  - ده نموذج شائع: الـ OUs بتقلد هيكل الشركة عشان تطبق سياسات مختلفة لكل قسم.

### الفرق بين OUs و Security Groups (مهم جدًا تفهميه)

| الغرض                  | Organizational Units (OUs)                  | Security Groups                           |
|-------------------------|---------------------------------------------|-------------------------------------------|
| الاستخدام الرئيسي     | تطبيق سياسات (Group Policy) على مجموعات   | منح صلاحيات على موارد (files, printers) |
| عدد العضوية            | يوزر/جهاز ينتمي لـ OU واحدة بس             | يوزر/جهاز ينتمي لعدة مجموعات            |
| مثال                    | قسم المبيعات عنده سياسات مختلفة عن الـ IT | مجموعة "HR Files Access" تدخل على مجلد معين |

### مصادر وأدوات مفيدة (2026)
- **TryHackMe**: كمل الـ room Active Directory Basics، وبعدها Breaching AD و Exploiting Active Directory.
- **Microsoft Learn** (مجاني): "Manage users and groups in Active Directory".
- **PowerShell** (أهم أداة في الـ AD):
  - Get-ADUser -Filter *  
  - New-ADUser  
  - Get-ADGroupMember "Domain Admins"
- **أدوات خارجية مجانية**:
  - ADRecon (للـ enumeration في الـ pentest).
  - BloodHound (لرسم العلاقات بين يوزرز ومجموعات – قوي جدًا في الـ red teaming).
